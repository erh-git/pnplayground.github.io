Symmetric Encryption

May 5, 2023

openssl encryption#

Encryption#

Openssl can be used for Symmetric Encryption:

echo Hello | openssl enc

Nothing happened because an algorithm wasn’t specified

List all possible algorithms using -list:

echo Hello | openssl enc -list

Encrypting with AES-128:

echo Hello | openssl enc -aes-128-cbc

Password requested because Symmetric Encryption requires Secret Key

Provide Password in Command echo Hello | openssl enc -aes-128-cbc -pass pass:SECRETKEY

On versions of OpenSSL after 1.0, you get this ominous warning: *** WARNING : deprecated key derivation used. Using -iter or -pbkdf2 would be better.

AES-128 requires a 128 bit key, yet we provided only “SECRETKEY". OpenSSL is taking the provided password and turning it into a Key in the required format… this is known as a KDF, or a Key Deriviation Function

The default algorithm OpenSSL uses is no longer considered secure, so OpenSSL is recommending we use more Iterations, or PBKDF2 (password-based key derivation function #2) echo Hello | openssl enc -aes-128-cbc -pass pass:SECRETKEY -pbkdf2

Result is Binary encoded, doesn’t display properly in ASCII

Format output in Base64 echo Hello | openssl enc -aes-128-cbc -pass pass:SECRETKEY -a

OpenSSL automatically adds a Salt with each encryption: echo Hello | openssl enc -a -aes-128-cbc -pass pass:SECRETKEY

Remove Salt with –nosalt option: echo Hello | openssl enc -a -aes-128-cbc -pass pass:SECRETKEY -nosalt

Changing the Message or Secret Key will change the resulting Cipher Text: echo Hello | openssl enc -a -nosalt -aes-128-cbc -pass pass:SECRETkey echo Hello | openssl enc -a -nosalt -aes-128-cbc -pass pass:secretKEY echo hi | openssl enc -a -nosalt -aes-128-cbc -pass pass:SECRETKEY echo HI | openssl enc -a -nosalt -aes-128-cbc -pass pass:SECRETKEY echo Hello | openssl enc -a -nosalt aes-128-cbc -pass pass:SECRETKEY

“Hello” encrypted using AES-128, without SALT, with key “SECRETKEY” echo Hello | openssl enc -a -nosalt -aes-128-cbc -pass pass:SECRETKEY

Cipher Text: fbxf/7u62sXeKSKU4K94dA==

Decryption#

OpenSSL can also do Decryption using -d option: echo "fbxf/7u62sXeKSKU4K94dA==" | openssl enc -aes-128-cbc -a -nosalt -d -pass pass:SECRETKEY

If the Cipher Text or the Symmetric Key are different, Decryption won’t work: echo "AAAf/7u62sXeKSKU4K94dA==" | openssl enc -aes-128-cbc -a -nosalt -d -pass pass:SECRETKEY echo "fbxf/7u62sXeKSKU4K94dA==" | openssl enc -aes-128-cbc -a -nosalt -d -pass pass:SECRETkey

OpenSSL can also encrypt and decrypt files: openssl enc -aes-128-cbc -a -nosalt -pass pass:SECRETKEY -in mobydick.txt -out mobydick.txt_ciphertext openssl enc -aes-128-cbc -a -nosalt -pass pass:SECRETKEY -d -in mobydick.txt_ciphertext -out mobydick.txt_plaintext

Examples#

$ echo "Any Questions?" | openssl enc  -aes-128-cbc  -pbkdf2 -a  -nosalt  -pass pass:PRACNET
